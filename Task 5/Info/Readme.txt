Описание архитектурного решения: использование Jwt токена с двумя сервисами: один для регистрации и логина (выдачи токена),
 второй - для получения и обновления информации о юзере.
За счет ингрессов пути доступны по одному адресу, нужная команда отправляется на свой сервис.

1. Команды установки:
cd "./authApp"
helm install auth-app ./hello-chart

cd "./testApp"
helm install test-app ./hello-chart

2. Я использовал неймспейс auth, хотя вроде бы не должно быть привязки
3. Тесты postman лежат в соответствующей папке 
    - cd "./postman"
    - newman run "Token auth tests.postman_collection.json"
